Listen on your phone (GitHub Pages + Mac mini)

The station page is static HTML. Your phone loads it over HTTPS, so the stream and API URLs must not be blocked as “mixed content.”

1. HTTPS stream (Tailscale Serve)

On the Mac mini, forward Icecast (port 8000) through Tailscale’s HTTPS proxy so the mount is an https:// URL (only on your tailnet unless you use Funnel):

tailscale serve --bg http://127.0.0.1:8000

Then your stream may look like https://suhyun.tailf2c209.ts.net/stream (exact URL is shown by the serve command; use your machine’s MagicDNS name). Open the Tailscale app on the phone so you are on the same tailnet.

2. Now playing on the static site

GET /now-playing on port 8001 is usually http://, which the browser will not fetch from an https:// GitHub Page. Options:

  1. Leave API blank and place a now_playing.json next to index.html (updated by your sync from the feeder’s output/now_playing.json, e.g. into your bskthefirst.github.io repo), or
  2. Point “Now playing JSON” to a full https:// URL that you host, or
  3. Expose the API over HTTPS (reverse proxy or tunnel) and paste that base URL in connection settings.

3. Enable GitHub Pages

Repository Settings → Pages → Build and deployment: source Deploy from a branch with folder /docs on main, or use the root of a username.github.io repo and copy these files there. Your site URL will look like https://<user>.github.io/<repo>/.

4. Bookmark on your phone

Open the deployed index, expand connection, paste the HTTPS stream URL, save, then add the page to the home screen or bookmarks. Optional: share a link with query parameters, e.g. ?stream=https%3A%2F%2F…%2Fstream&np=now_playing.json.
